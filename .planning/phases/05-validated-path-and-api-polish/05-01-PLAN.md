<objective>
Implement the validated conversion path: when `validate=True` is set on `ArrowModelConverter`, each row is serialized to JSON bytes in Rust via `serde_json` and passed to Pydantic's `model_validate_json` for full validation.

Purpose: Users dealing with untrusted data can opt into full Pydantic validation (type coercion, constraint checking, custom validators) while still benefiting from Rust-speed Arrow buffer extraction. This completes the dual-path architecture: fast path (model_construct, no validation) and validated path (serde_json + model_validate_json).

Output: New Rust functions `convert_record_batch_validated`/`convert_table_validated`, `extract_json_value` on all ColumnExtractor variants, Python-side branching in `convert()`, and comprehensive tests.
</objective>

    - `Int8/Int16/Int32/Int64`: Check `is_null(row)` -> `Value::Null`. Otherwise `Value::Number(Number::from(arr.value(row)))`.
    - `UInt8/UInt16/UInt32/UInt64`: Same pattern with `Number::from`.
    - `Float32/Float64`: Check `is_null(row)` -> `Value::Null`. Otherwise check `arr.value(row).is_nan() || arr.value(row).is_infinite()` -> `Value::Null` (Pitfall 5 from research). Otherwise `Value::Number(Number::from_f64(arr.value(row) as f64).unwrap_or(Value::Null))`. Note: `Number::from_f64` returns `Option` so use `.map(Value::Number).unwrap_or(Value::Null)`.
    - `Boolean`: `Value::Null` or `Value::Bool(arr.value(row))`.
    - `Utf8/LargeUtf8`: `Value::Null` or `Value::String(arr.value(row).to_owned())`.
    - `Date32`: `Value::Null` or use `arr.value_as_date(row)` then `Value::String(d.format("%Y-%m-%d").to_string())`. Need `use chrono::NaiveDate;` for format method.
    - `TimestampNaive`: Extract naive datetime same as extract_value does (via `value_as_datetime`), then format as ISO 8601 string: `dt.format("%Y-%m-%dT%H:%M:%S%.f").to_string()`. Return `Value::String(...)`.
    - `TimestampAware`: Same as TimestampNaive for the datetime extraction (timezone is part of the schema, not the value). Format as ISO 8601 with timezone offset. Since we have the timezone string from the Arrow schema (stored in `DataType::Timestamp(unit, Some(tz_str))`), we need to pass it through. **However**, the `TimestampAware` variant stores a `PyObject` (ZoneInfo), not the tz string. For JSON serialization, we need the tz string. **Solution**: Add a `String` field to `TimestampAware` variant to store the timezone string alongside the PyObject. Update `prepare_extractor` to pass `tz_str.to_string()` as the 4th field. Then in `extract_json_value`, format as `"YYYY-MM-DDTHH:MM:SS.ffffff+HH:MM"` -- but since Pydantic's `model_validate_json` also accepts naive datetime strings and applies timezone from the model annotation, the simplest approach is to just format the UTC datetime as ISO 8601 with `+00:00` suffix. **Simpler alternative**: Since the raw Arrow timestamp value is always in UTC (Arrow spec), format the naive datetime and append the tz string. Actually, the simplest correct approach: format as naive ISO 8601 string. Pydantic's `model_validate_json` accepts naive datetime strings for both naive and aware datetime fields (it will make them aware based on model config). Use `dt.format("%Y-%m-%dT%H:%M:%S%.f").to_string()` and let Pydantic handle timezone application.
    - `Duration`: Convert to ISO 8601 duration string. Use the `timedelta_to_iso8601` helper from research. Extract the raw value via the same `value_as_duration` call, then convert the `chrono::TimeDelta` to ISO 8601 format `PxDTxHxMxS`. Implement a helper function `fn timedelta_to_iso8601(td: &chrono::TimeDelta) -> String` in extract.rs using the algorithm from 05-RESEARCH.md lines 288-328.

    - `Struct`: `Value::Null` or build `Value::Object(map)` by iterating children and calling `extract_json_value` recursively. Use the field name strings from the children vec. **Important**: The Struct variant stores `Py<PyString>` for field names. For JSON keys, convert to Rust String. Call `field_name.bind(py).to_str()?.to_owned()` to get the Rust string. This means `extract_json_value` needs a `py: Python<'_>` parameter for the Struct and TimestampAware variants.
    - `Null`: Always return `Value::Null`.

    Add `use pyo3::types::PyBytes;` and `use serde_json;` to imports in lib.rs.

    Add `#[pyfunction] fn convert_record_batch_validated(py, batch: PyRecordBatch, model_cls: Bound<'_, PyAny>, field_specs: Vec<(usize, String, Option<PyObject>)>) -> PyResult<PyObject>` that:
    1. Decomposes field_specs into col_indices, field_names, nested_models (same as convert_record_batch)
    2. Pre-unpacks dictionary columns via `unpack_columns`
    3. Builds extractors via `prepare_extractor` (same as convert_record_batch)
    4. For each row: builds `serde_json::Map<String, Value>` by calling `extractor.extract_json_value(py, row)` for each column, inserting with field_name as key
    5. Calls `serde_json::to_vec(&Value::Object(map))` to produce JSON bytes
    6. Creates `PyBytes::new(py, &json_bytes)`
    7. Calls `model_cls.call_method1("model_validate_json", (py_bytes,))?` to get validated instance
    8. Collects into `Vec<PyObject>`, returns as `PyList`

    Add `#[pyfunction] fn convert_table_validated(py, table: PyTable, model_cls, field_specs)` following the same pattern as `convert_table` but using the validated row loop from above. Iterate all batches, build JSON per row, call model_validate_json.

    **Part C: Update Python `convert()` to branch on `self._validate`**

    In `src/arrowdantic/__init__.py`, modify the `convert()` method to check `self._validate`. When True, call `_core.convert_record_batch_validated` or `_core.convert_table_validated` instead of the non-validated versions. The field_specs are the same (already resolved by `_resolve_columns`).

    ```python
    def convert(self, data: pa.RecordBatch | pa.Table) -> list[BaseModel]:
        field_specs = self._resolve_columns(data.schema)
        if hasattr(data, "to_batches"):
            if self._validate:
                return _core.convert_table_validated(data, self._model_class, field_specs)
            return _core.convert_table(data, self._model_class, field_specs)
        else:
            if self._validate:
                return _core.convert_record_batch_validated(data, self._model_class, field_specs)
            return _core.convert_record_batch(data, self._model_class, field_specs)
    ```
