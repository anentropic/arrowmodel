## Tech Debt

### Phase 6: Support All PyArrow Types

1. **IntervalYearMonth and IntervalDayTime subtypes have zero test coverage.** Rust implementation exists (`extract.rs` lines 624-667 for `extract_value`, 1141-1165 for `extract_json_value`) but no fixtures (`interval_ym_batch`, `interval_dt_batch`) or test classes were created. Only `IntervalMonthDayNano` is tested.

2. **Validated path untested for 7 types.** The following types have `extract_json_value` implemented and the fast path tested, but no `validate=True` test: Decimal256, Time64, LargeBinary, FixedSizeBinary, BinaryView, RunEndEncoded, Union.

### Phase 5: Validated Path & API Polish

3. **`iter_arrow(validate=True)` untested.** The convenience wrapper accepts `validate` and passes it through, but only fast-path iteration is tested. The `iter()` method validated path works (tested), so this is a coverage gap in the convenience wrapper only.

4. **`from_arrow()` lacks `validate` parameter.** `iter_arrow()` supports `validate=True` but `from_arrow()` does not. API asymmetry — users cannot do validated one-shot conversion without constructing `ArrowModelConverter` directly. May be by design (API-03 spec doesn't mention validation).

### Total: 4 items across 2 phases
