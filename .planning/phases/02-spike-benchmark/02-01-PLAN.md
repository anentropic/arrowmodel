The function body must:
  a. Call `batch.into_inner()` to get the `RecordBatch`
  b. Intern field names once using `PyString::intern(py, &name)` for each name (per FAST-03, research Pattern 5). Collect into `Vec<Bound<'_, PyString>>`.

  e. Row loop (`for row in 0..num_rows`):
     - Create `PyDict::new(py)` for kwargs
     - For each `(extractor, interned_name)` pair: call `extractor.extract_value(py, row)?` and set on dict via `kwargs.set_item(interned_name, value)?`

  f. Convert results Vec to `PyList::new(py, &results)` and return as `PyObject`

CRITICAL anti-patterns to avoid:
- Do NOT use `call_method1` for model_construct (Pitfall 3 -- it passes dict as positional arg, not kwargs)
- Do NOT use `intern!()` macro for field names (Pitfall 1 -- requires compile-time literals)
- Do NOT call `value(row)` without checking `is_null(row)` first (Pitfall 2 -- undefined data at null indices)
- Do NOT match on DataType inside the row loop (anti-pattern -- match once per column before loop)

**Replace the contents of `src/arrowdantic/__init__.py`** with:

```python
"""Arrowdantic: dict-free conversion from Arrow buffers to Pydantic model instances."""

from __future__ import annotations

from typing import TYPE_CHECKING

from arrowdantic import _core as _core

if TYPE_CHECKING:
    import pyarrow as pa
    from pydantic import BaseModel

__all__ = ["ArrowModelConverter", "_core"]


class ArrowModelConverter:
    """Convert Arrow RecordBatch data to Pydantic model instances.

    Cross-references Arrow schema against Pydantic model fields at
    construction time (field names stored) and at convert() time
    (column indices resolved per-batch schema). Uses model_construct
    for zero-validation fast path.

    Per SCHEMA-01: Cross-references Arrow schema against Pydantic model fields.
    Per SCHEMA-02: Schema mapping (field names list) compiled once at init, reused.
    Per API-01: Constructor accepts model class and optional validate flag.
    """

    def __init__(
        self,
        model_class: type[BaseModel],
        *,
        validate: bool = False,
    ) -> None:
        self._model_class = model_class
        self._validate = validate
        # SCHEMA-01, SCHEMA-02: Extract field names once at init, reuse across batches
        self._field_names: list[str] = list(model_class.model_fields.keys())

    def convert(self, data: pa.RecordBatch) -> list[BaseModel]:
        """Convert an Arrow RecordBatch to a list of Pydantic model instances.

        Per API-02: Returns list[Model].
        Per INPUT-01: Accepts pyarrow RecordBatch.
        Per SCHEMA-01: Matches Arrow column names to stored Pydantic field names.

        Raises ValueError if a Pydantic field name is not found in the Arrow schema.
        """
        schema = data.schema
        col_indices: list[int] = []
        for field_name in self._field_names:
            try:
                col_idx = schema.get_field_index(field_name)
            except KeyError:
                msg = (
                    f"Arrow schema has no column named {field_name!r}. "
                    f"Available columns: {schema.names}"
                )
                raise ValueError(msg) from None
            col_indices.append(col_idx)

        return _core.convert_record_batch(
            data,
            self._model_class,
            col_indices,
            self._field_names,
        )
```

Key design decisions:
- Field names extracted once at `__init__` (SCHEMA-02: compiled once, reused)
- Column index resolution at `convert()` time because each batch may have different column order (per research Open Question 2)
- Uses `schema.get_field_index()` which raises KeyError if not found -- caught and re-raised as ValueError with helpful message
- `validate` parameter accepted but not used yet (Phase 5 VALID-01)
- TYPE_CHECKING guard for pyarrow import (it is optional at runtime)
