Add ColumnExtractor support for all scalar, temporal, and binary Arrow types not covered in Phases 2-5: Float16, Decimal128/256/32/64, Date64, Time32/64, Binary/LargeBinary/FixedSizeBinary, Utf8View/BinaryView.

Purpose: Extends arrowdantic to handle numeric precision types (decimals), time-of-day types, binary data, and the newer view-based string/binary layouts. These are the simpler "leaf" types that don't require container/recursive logic.

Output: Updated extract.rs with 14 new ColumnExtractor variants (both fast and validated paths), base64 crate added to Cargo.toml, comprehensive test coverage in test_extended_types.py.
