    1. Update `rust/Cargo.toml`:
       - Change pyo3 features to `["extension-module", "chrono"]` to enable automatic NaiveDate/NaiveDateTime/TimeDelta -> PyDate/PyDateTime/PyDelta conversion
       - Add `arrow-cast = "58"` dependency for dictionary array unpacking via `arrow_cast::cast()`

    2. Extend `ColumnExtractor` enum in `rust/src/extract.rs` with these new variants:
       ```
       Date32(&'a arrow_array::Date32Array)
       TimestampNaive(&'a dyn Array, TimeUnit)
       TimestampAware(&'a dyn Array, TimeUnit, PyObject)  // PyObject = cached ZoneInfo
       Duration(&'a dyn Array, TimeUnit)
       Dictionary(Box<ColumnExtractor<'a>>, ArrayRef)  // holds unpacked array + extractor
       Null
       ```

       Note on Dictionary lifetime: `arrow_cast::cast()` returns an owned `ArrayRef`. Store it in the enum variant as an owned `ArrayRef`. The `ColumnExtractor` inside the `Box` borrows from this owned array. To handle this, use the pre-unpack approach: in `prepare_extractor`, unpack the dictionary first, store the owned `ArrayRef` in a separate collection (or change the Dictionary variant to hold the owned array), then create the inner extractor. The simplest approach is to change `ColumnExtractor` to hold `ArrayRef` for Dictionary and re-downcast in `extract_value`.

       Actually, the cleanest approach for Dictionary: in `convert_record_batch` (lib.rs), detect dictionary columns BEFORE building extractors, cast them to value type, and replace the column reference. This way `prepare_extractor` never sees Dictionary -- it sees the unpacked value type. Add a helper function `unpack_dictionary_columns` that takes the RecordBatch columns + schema and returns a `Vec<ArrayRef>` of potentially-unpacked columns.

    3. Add new imports to `extract.rs`:
       ```rust
       use arrow_array::{
           cast::{as_primitive_array, as_boolean_array, as_string_array, as_largestring_array},
           types::{Date32Type, TimestampSecondType, TimestampMillisecondType,
                   TimestampMicrosecondType, TimestampNanosecondType,
                   DurationSecondType, DurationMillisecondType,
                   DurationMicrosecondType, DurationNanosecondType},
           Date32Array, Array, BooleanArray, StringArray, LargeStringArray,
       };
       use arrow_schema::{DataType, TimeUnit};
       use pyo3::types::{PyDateTime, PyTzInfoAccess};
       ```

    4. Update `prepare_extractor` match arms:
       - `DataType::Date32` => downcast to `Date32Array`, return `ColumnExtractor::Date32(arr)`
       - `DataType::Timestamp(unit, None)` => return `ColumnExtractor::TimestampNaive(col, *unit)`
       - `DataType::Timestamp(unit, Some(tz))` => import `zoneinfo.ZoneInfo`, create `ZoneInfo(tz.as_ref())`, cache as PyObject, return `ColumnExtractor::TimestampAware(col, *unit, tz_obj)`
       - `DataType::Duration(unit)` => return `ColumnExtractor::Duration(col, *unit)`
       - `DataType::Null` => return `ColumnExtractor::Null`
       - `DataType::Dictionary(_, _)` => should not reach here if pre-unpacked in lib.rs

    5. Implement `extract_value` for each new variant:

       **Date32:** Use `arr.value_as_date(row)` which returns `Option<chrono::NaiveDate>`. With pyo3 chrono feature, call `.into_pyobject(py)?.into_any().unbind()`. Check `arr.is_null(row)` first.

       **TimestampNaive:** Match on TimeUnit to downcast to the correct timestamp array type (TimestampSecondType, TimestampMillisecondType, TimestampMicrosecondType, TimestampNanosecondType). Use `value_as_datetime(row)` which returns `Option<chrono::NaiveDateTime>`. Convert via `into_pyobject`. TEMP-05: Nanosecond truncation happens automatically because chrono's NaiveDateTime stores microsecond precision.

       **TimestampAware:** Same datetime extraction as TimestampNaive, but then construct `PyDateTime::new(py, year, month, day, hour, min, sec, microsecond, Some(&tz_obj.bind(py).downcast::<PyTzInfo>()?))`. Extract components from the NaiveDateTime: `.year()`, `.month() as u8`, `.day() as u8`, `.hour() as u8`, `.minute() as u8`, `.second() as u8`, `(.nanosecond() / 1000) as u32` for microseconds. The tz_obj is the cached `zoneinfo.ZoneInfo` PyObject.

       **Duration:** Match on TimeUnit, downcast to appropriate DurationType array, use `value_as_duration(row)` which returns `Option<chrono::TimeDelta>`. Convert via `into_pyobject`.

       **Null:** Always return `py.None()`. Do NOT check `is_null()` -- NullArray has no physical null buffer so `is_null()` returns false (Pitfall 1 from research).

    6. Update `convert_record_batch` and `convert_table` in `lib.rs` to pre-unpack dictionary columns:
       - Before building extractors, iterate columns and check if any have `DataType::Dictionary(_, value_type)`
       - If so, use `arrow_cast::cast(col, value_type.as_ref())` to unpack
       - Store unpacked arrays in a `Vec<ArrayRef>` so their lifetime extends through the extractor usage
       - Pass unpacked column references to `prepare_extractor` instead of original dictionary columns

    7. Make `prepare_extractor` accept `Python<'_>` as an additional parameter (needed for timezone-aware timestamps to import zoneinfo). Update the signature:

       ```rust
       ) -> PyResult<ColumnExtractor<'a>>
       ```
